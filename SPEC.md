# SPEC.md — 機能仕様書

## 1. 製品概要

### プロダクト名
スキル分析・育成計画 レポートジェネレーター

### 目的
IT部門のチームリーダーが、スキル分析・育成計画データをブラウザ上のフォームに入力し、
整形されたHTMLレポートをワンクリックで生成・ダウンロードできるツール。

### 配布形態
単一HTMLファイル（`index.html`）をファイルサーバーまたはメールで配布。
受け取った担当者はブラウザで開くだけで使用できる。

---

## 2. ユーザーフロー

```
[1] index.html をブラウザで開く
        ↓
[2] Step 1: 基本情報を入力
        ↓
[3] Step 2: 予定案件を入力（行を追加しながら）
        ↓
[4] Step 3: 育成方針を入力
        ↓
[5] Step 4: メンバー別育成目標を入力
        ↓
[6]「レポートを生成」ボタンを押す
        ↓
[7] 画面下部にHTMLレポートがプレビュー表示される
        ↓
[8]「HTMLをダウンロード」でファイル保存
        ↓
[9] 保存したHTMLをブラウザで開いて確認・印刷・共有
```

---

## 3. 画面仕様

### 3-1. アプリヘッダー
- タイトル: 「スキル分析・育成計画 レポートジェネレーター」
- サブテキスト: 「フォームに入力してレポートを生成 → HTMLファイルとしてダウンロード（サーバー不要）」
- 配色: グリーン（`#02993B`）をメインカラー、オレンジ（`#F39346`）をアクセントカラーとして使用

### 3-2. ステップナビゲーション
- 4ステップをタブ形式で表示
- 状態: `active`（現在）/ `done`（完了）/ 未着手
- ステップ間は自由に行き来可能（上部タブクリックでジャンプ可）

## 4. フォーム仕様

### Step 1 — 基本情報

| フィールドID | ラベル | 入力型 | 必須 | 備考 |
|-------------|--------|--------|------|------|
| `f-year` | 年度 | text | ✅ | 例: 2025年度 |
| `f-team` | 部門 / チーム名 | text | ✅ | 例: IT基盤部 / クラウド基盤チーム |
| `f-leader` | リーダー名 | text | ✅ | |
| `f-period` | 対象期間 | text | — | 例: 2025〜2029年度 |
| `f-date` | 作成日 | date | — | デフォルト: 当日 |
| `f-theme` | 重点テーマ | text | — | 例: クラウド移行、内製化推進 |
| `f-scope` | 担当領域・役割概要 | textarea | — | 最大300文字推奨 |

**バリデーション**: `f-year` `f-team` `f-leader` のいずれかが空の場合、
Step 2 への遷移をブロックしてエラーメッセージを表示する。

---

### Step 2 — 予定案件（繰り返し行テーブル）

初期行: 1行（空）。「＋ 案件を追加」ボタンで行を追加。

| 列ID | ラベル | 入力型 | 必須 | 選択肢 |
|------|--------|--------|------|--------|
| `pnm{id}` | 案件名 | text | ✅ | — |
| `pov{id}` | 概要 | text | — | — |
| `ptm{id}` | 実施時期 | text | — | 例: 2025年度 |
| `pkn{id}` | 区分 | select | — | 更改 / 新規 / 移行 / 運用改善 / 内製化 |
| `psk{id}` | 必要スキル領域 | text | — | カンマ区切りで複数入力可 |
| `ppr{id}` | 重要度 | select | — | 高 / 中 / 低 |

- 案件名が空の行はレポート生成時に除外する
- 最大件数制限なし（UI上の制約のみ）

---

### Step 3 — 育成方針

| フィールドID | ラベル | 入力型 | 必須 |
|-------------|--------|--------|------|
| `f-focus` | 重点スキル領域 | text | — |
| `f-target` | 育成対象層 | select | — |
| `f-vision` | 目指す人材像 | text | — |
| `f-gap` | 現状の課題・ギャップ | textarea | — |
| `f-policy` | 育成方針（自由記述） | textarea | — |

`f-target` の選択肢: 若手中心（〜5年目）/ 中堅中心（5〜10年目）/ 次期リーダー候補 / 全層

---

### Step 4 — メンバー別育成目標（繰り返し行テーブル）

初期行: 1行（空）。「＋ メンバーを追加」ボタンで行を追加。

| 列ID | ラベル | 入力型 | 必須 | 備考 |
|------|--------|--------|------|------|
| `mnm{id}` | 氏名 | text | ✅ | — |
| `msk{id}` | 育成対象スキル | text | — | — |
| `mcl{id}` | 現在Lv | select | — | 1〜5 |
| `mgl{id}` | 目標Lv | select | — | 1〜5 |
| `mtm{id}` | 目標時期 | text | — | 例: 2026年度 |
| `mms{id}` | 主な育成施策 | text | — | 読点・カンマ区切りで複数入力 → チップ表示 |
| `mnt{id}` | 備考 | text | — | — |

- 氏名が空の行はレポート生成時に除外する
- 施策フィールドは `、` `,` `，` で分割してタグチップとして表示

## 5. レポート出力仕様

### 5-1. 出力形式
- スタンドアロンHTMLファイル（外部リソース参照なし）
- 文字コード: UTF-8
- ファイル名: `スキル育成計画_{チーム名}_{年度}.html`

### 5-2. レポート構成

| セクション | 内容 |
|-----------|------|
| 表紙 | 年度・チーム名・リーダー・対象期間・作成日・担当領域概要 |
| 1. 予定案件一覧 | テーブル形式（案件名・概要・時期・区分・スキル・重要度バッジ） |
| 2. 育成方針 | カードグリッド（重点領域・対象層・人材像・課題・方針） |
| 3. メンバー別育成目標 | メンバーカード（アバター・Lvビジュアル・施策チップ） |
| 参考. スキルレベル定義 | Lv1〜5の定義テーブル |
| フッター | チーム名・年度・生成日時 |

### 5-3. デザイン仕様

| 要素 | 仕様 |
|------|------|
| メインカラー | グリーン `#02993B` |
| アクセントカラー | オレンジ `#F39346` |
| 背景色 | ホワイト `#FFFFFF` |
| 文字色 | `#212121` |
| フォント | `"Noto Sans JP", "Yu Gothic UI", "Hiragino Sans", "Meiryo", sans-serif` |
| 重要度バッジ | 高=オレンジ系 / 中=黄系 / 低=グリーン系 |
| Lvインジケーター | 塗りつぶし済みドット（グリーン）× 空ドット（グレー）× 5個 |
| メンバーアバター | 名前から自動生成する2文字イニシャル＋カラー円 |
| 印刷対応 | `@media print` でブレーク制御・背景省略 |

### 5-4. XSS対策
すべてのユーザー入力値は `escapeHTML()` でサニタイズしてからHTMLに埋め込む。

## 6. ダウンロード仕様

```javascript
// Blob URLを使ったダウンロード
const blob = new Blob([reportHTML], { type: 'text/html;charset=utf-8' });
const a    = document.createElement('a');
a.href     = URL.createObjectURL(blob);
a.download = `スキル育成計画_${team}_${year}.html`;
a.click();
```

## 7. スコープ外（v1.0）

以下は意図的に含めない:

- データの永続化（localStorage / IndexedDB への保存）
- PDF出力
- Excel / CSV インポート・エクスポート
- 複数チームの一括管理
- ユーザー認証
- サーバーへのデータ送信
- AIによるレポート文章生成
