# テストケース — 一時保存・読み込み機能

対象バグ修正:
- **修正1**: ID生成を `Date.now()` から連番カウンター（`genId()`）に変更（複数行復元時のID重複を解消）
- **修正2**: 保存時の空行フィルタリングを廃止し、空行も含めて完全復元するよう変更

---

## テスト実施手順

ブラウザで `index.html` を直接開き、以下の各ケースを順番に手動で確認する。
各ケースは独立しているため、ケースをまたぐ場合はページをリロードしてから実施する。

---

## TC-01: 基本的な保存・読み込み（正常系）

**目的**: Step1〜4を入力したデータが保存・復元されること

**手順**:
1. Step1に以下を入力する
   - 年度: `2026年度`
   - 部門: `デジタル基盤グループ・クラウド推進チーム`
   - リーダー名: `山田 太郎`
   - 対象期間: `2026年度〜2028年度`
   - 重点テーマ: `クラウド移行`
2. 「次へ」でStep2へ進み、案件を1件入力する
   - 案件名: `AWS移行プロジェクト`
   - 概要: `オンプレからAWSへの移行`
   - 重要度: `高`
3. Step3へ進み、育成方針を1件入力する
   - 重点スキル: `クラウド推進・クラウド基盤`
   - 目指す人材像: `AWSアーキテクト`
4. Step4へ進み、メンバーを1件入力する
   - 氏名: `鈴木 花子`
   - 現在Lv: `2`、目標Lv: `4`
5. 「保存」ボタンを押してJSONをダウンロードする
6. ページをリロードする
7. 「読み込み」ボタンを押してJSONを選択する

**期待結果**:
- [ ] Step1の全フィールドが復元されている
- [ ] Step2の案件名「AWS移行プロジェクト」が復元されている
- [ ] Step3の重点スキル・人材像が復元されている
- [ ] Step4のメンバー「鈴木 花子」と Lv が復元されている
- [ ] 読み込み後は Step1 が表示されている

## TC-02: 複数行の正確な復元【修正1の検証】

**目的**: 複数の案件・メンバーが1件ずつ正確に復元されること（ID重複バグの回帰テスト）

**手順**:
1. Step2で案件を3件追加し、それぞれ別の名前を入力する
   - 案件A: `プロジェクトAAA`
   - 案件B: `プロジェクトBBB`
   - 案件C: `プロジェクトCCC`
2. Step4でメンバーを3件追加し、それぞれ別の氏名を入力する
   - メンバー1: `田中 一郎`
   - メンバー2: `佐藤 二郎`
   - メンバー3: `高橋 三郎`
3. 「保存」ボタンを押す
4. ページをリロードして読み込む

**期待結果**:
- [ ] Step2に案件が3件表示される
- [ ] 案件の順序が保持されている（A→B→Cの順）
- [ ] 各案件の名前が正しく対応している（1件目がAAA、2件目がBBB、3件目がCCC）
- [ ] Step4にメンバーが3件表示される
- [ ] 各メンバーの名前が正しく対応している（1件目が田中、2件目が佐藤、3件目が高橋）
- [ ] データの混在・上書きが起きていない（例: 全件が「プロジェクトCCC」になっていない）

## TC-03: 空行の保存・復元【修正2の検証】

**目的**: 名前が未入力の行も保存され、読み込み後に空行として復元されること

**手順**:
1. Step2で「＋ 案件を追加」を2回押し、計3行にする（初期1行 + 追加2行）
   - 1行目: 案件名「テスト案件」のみ入力
   - 2行目: 何も入力しない（空行）
   - 3行目: 何も入力しない（空行）
2. Step4で「＋ メンバーを追加」を1回押し、計2行にする
   - 1行目: 氏名「田中 太郎」のみ入力
   - 2行目: 何も入力しない（空行）
3. 「保存」ボタンを押す
4. ページをリロードして読み込む

**期待結果**:
- [ ] Step2に案件が3行表示される（入力済み1行 + 空行2行）
- [ ] 1行目の案件名に「テスト案件」が入っている
- [ ] 2・3行目は空欄のまま表示される
- [ ] Step4にメンバーが2行表示される（入力済み1行 + 空行1行）
- [ ] 1行目の氏名に「田中 太郎」が入っている
- [ ] 2行目は空欄のまま表示される

## TC-04: 空行はレポートに出力されない【修正2の副作用検証】

**目的**: 空行が保存・復元されても、レポートには出力されないこと

**前提**: TC-03 の読み込み完了後の状態から続けて実施する

**手順**:
1. Step1に必須項目（年度・部門・リーダー名）を入力して「次へ」で Step4 まで進む
2. 「レポートを生成」ボタンを押す

**期待結果**:
- [ ] レポートの「予定案件 一覧」に「テスト案件」が1件だけ表示される（空行2件は出力されない）
- [ ] レポートの「メンバー別 育成目標」に「田中 太郎」が1件だけ表示される（空行1件は出力されない）
- [ ] レポートに空のカードや空白のセクションが混入していない

## TC-05: 空行のみ・0件の保存・復元

**目的**: 行を全て削除した状態、または何も追加していない状態の保存・復元が正常に動作すること

**手順**:
1. Step2で初期の1行を「✕ 削除」ボタンで削除する（案件0件の状態）
2. Step3で初期の1行を「✕ 削除」ボタンで削除する（育成方針0件の状態）
3. Step4で初期の1行を「✕ 削除」ボタンで削除する（メンバー0件の状態）
4. 「保存」ボタンを押す
5. ページをリロードして読み込む

**期待結果**:
- [ ] エラーが表示されない
- [ ] Step2 に初期空行が1件表示される（0件の場合は初期化で1件生成される仕様）
- [ ] Step3 に初期空行が1件表示される
- [ ] Step4 に初期空行が1件表示される

## TC-07: 不正なファイル・バージョン不一致

**目的**: 不正なファイルを読み込んだ際にエラーが表示され、データが壊れないこと

**手順A（JSONでないファイルを選択）**:
1. 「読み込み」ボタンを押し、`.txt` や `.html` ファイルを選択する
2. エラーバナーが表示されることを確認する
3. 現在の入力データが変化していないことを確認する

**手順B（別アプリのJSONを選択）**:
1. 任意の内容のJSONファイルを用意する（例: `{"key":"value"}`）
2. 「読み込み」ボタンで選択する

**期待結果**:
- [ ] 手順A: 「JSONファイルの解析に失敗しました」エラーが表示される
- [ ] 手順B: 「このアプリ用のデータファイルではありません」エラーが表示される
- [ ] いずれの場合も既存の入力データは変化していない

## TC-08: 同じファイルの連続読み込み

**目的**: 一度読み込んだファイルを再度選択できること（ファイル input のリセット確認）

**手順**:
1. JSONを保存してリロード後に読み込む
2. 確認ダイアログで「OK」を押して復元する
3. 再度「読み込み」ボタンを押し、同じJSONファイルを選択する

**期待結果**:
- [ ] 2回目の読み込みでも確認ダイアログが表示される
- [ ] 2回目もデータが正しく復元される（同じファイルを再選択できる）

## TC-09: RTE（リッチテキスト）フィールドの保存・復元

**目的**: 書式付きテキスト（太字・箇条書き・文字色）が保存・復元されること

**手順**:
1. Step1「担当領域・役割概要」に以下を入力する
   - テキスト「クラウド推進」を太字にする
   - 箇条書きで2項目入力する
   - 1項目の文字色を赤に変更する
2. Step3「現状の課題・ギャップ」に書式付きテキストを入力する
3. 「保存」して読み込む

**期待結果**:
- [ ] 「担当領域・役割概要」の太字・箇条書き・文字色が復元されている
- [ ] 「現状の課題・ギャップ」の書式が復元されている
- [ ] 生成したレポートにも書式が反映されている
