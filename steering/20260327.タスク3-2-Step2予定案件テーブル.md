## 実装内容

### HTML
- 7列テーブル: 案件名（必須）/ 概要 / 実施時期 / 区分 / 必要スキル領域 / 重要度 / 削除

### CSS（新規追加）
- `.tbl-wrap`: 横スクロール対応、角丸ボーダー
- `.tbl-wrap table`: `white-space: nowrap` で列幅を保持
- テーブル行ホバーで薄いグレー背景
- `.add-btn`: グリーン薄背景・破線ボーダー
- `.del-btn`: 赤系薄背景
- `.btn-gen`: オレンジ（アクセントカラー）- 次タスクで使用

### JS（新規追加）
- `addProj()`: `Date.now()` をIDとして行追加、区分／重要度はセレクト
- `delRow(arr, id, rowId)`: 配列から削除 + DOM削除（案件・メンバー共通）
- 起動時 `init()` で `addProj()` を1回呼んで初期行を挿入

## 設計上の決定事項

| 決定内容 | 選択理由 |
|---------|---------|
| `white-space: nowrap` をテーブルに設定 | 列が折り返さず横スクロールで表示されるためレイアウトが崩れない |
| `delRow` を汎用関数に設計 | 案件・メンバーの両テーブルで共有できる |
| `.btn-gen`（オレンジ）を今タスクで定義 | Step4で使うが、テーブル系CSSとまとめて定義しておく |
| 初期行を `addProj()` で挿入 | 空テーブルよりもすぐ入力できる状態が UX 的に望ましい |
