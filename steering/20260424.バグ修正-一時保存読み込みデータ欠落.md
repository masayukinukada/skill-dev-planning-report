# バグ修正 — 一時保存・読み込み時のデータ欠落

**作業日**: 2026-04-24
**対象ファイル**: `index.html`, `test-cases-save-load.md`

---

## 不具合の内容

一時保存したJSONを読み込むと、複数の案件・育成方針・メンバーのデータが欠落または混在する事象が発生していた。

---

## 原因

### 原因1: ID生成に `Date.now()` を使用していた

`addProj()` / `addPolicy()` / `addMember()` はそれぞれ `const id = Date.now()` で行IDを生成していた。
読み込み時の `restoreProjects()` 等はループ内でこれらを同期的に連続呼び出しするため、**同一ミリ秒内で複数の `Date.now()` が同じ値を返し、IDが重複**した。

IDが重複すると `document.getElementById('pnm' + id)` が常に最初の要素を返すため、
2件目以降のデータがすべて1件目の入力欄に上書きされ、最後の1件だけが残る状態になっていた。

### 原因2: `collect()` が空行をフィルタリングしていた

保存時の `collect()` は以下の条件で行を除外していた。
- 案件: `name` が空
- メンバー: `name` が空
- 育成方針: 全フィールドが空

このため、意図的に残した空行や未入力状態の行が保存されず、読み込み後に行数が減るケースがあった。

---

## 修正内容

### 修正1: ID生成を連番カウンターに変更

```javascript
// 追加
let _idSeq = 0;
function genId() { return ++_idSeq; }

// 変更前
const id = Date.now();
// 変更後
const id = genId();
```

`addProj()` / `addPolicy()` / `addMember()` の3関数すべてを変更。

### 修正2: 保存時の空行フィルタリングを廃止

`collect()` から `.filter()` を削除し、空行も含めてすべて保存・復元するよう変更。

レポート生成時（`buildProjects()` / `buildPolicy()` / `buildMembers()`）のみ、
各関数の冒頭で空行を除外するフィルタを適用することで、
**保存・復元は完全保持、レポート出力は意味のある行のみ**という動作に分離した。

```javascript
// buildProjects() 例
function buildProjects(projects) {
  const filled = projects.filter(p => p.name);  // レポート生成時のみ除外
  ...
}
```

---

## 設計上の決定事項

| 決定内容 | 選択理由 |
|---------|---------|
| IDを連番カウンターにする（リセットしない） | セッション中に一意であれば十分。リセット不要でシンプル |
| `collect()` からフィルタを完全に除去する | 保存と生成でロジックを分離し、collect() の責務を「全データ収集」に統一 |
| フィルタをレポート生成側（build*関数）に移動する | 空行がレポートに混入しないという既存の要件を維持するため |
