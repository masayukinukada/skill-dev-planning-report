# REQUIREMENT.md — 要件定義書

## 1. 背景・目的

### 背景
IT部門では毎年度、各チームのリーダーがスキル分析・育成計画を作成し、
所定のPowerPointテンプレートに記入したうえで上長に提出している。

現状の課題:
- テンプレートPPTの書式設定・レイアウト調整に時間がかかる
- チームによって記載レベルや形式がバラバラになりがち
- 作成したファイルの共有・管理が煩雑

### 目的
フォームに入力するだけで統一フォーマットのHTMLレポートを自動生成することで、
育成計画の作成工数を削減し、記載内容の標準化を図る。

---

## 2. ステークホルダー

| 役割 | 関係 | 主な関心事 |
|------|------|-----------|
| チームリーダー | 主要ユーザー | 入力のしやすさ、出力の見やすさ |
| 部門管理者 | レポート受領者 | フォーマットの統一、印刷品質 |
| IT企画担当（Masayuki） | オーナー | 展開・保守のしやすさ、セキュリティ |

## 3. 機能要件

### FR-01 ステップ式フォーム入力
- フォームは4ステップ構成とし、画面上部のタブで現在位置を示す
- ステップタブをクリックすることで任意のステップに直接移動できる
- 入力済みのステップには「完了」表示を付与する

### FR-02 基本情報入力（Step 1）
- 年度・部門名・リーダー名は必須項目とする
- 必須項目が未入力の場合、次ステップへの遷移をブロックしてエラーを表示する
- 作成日は当日の日付をデフォルト値として設定する

### FR-03 予定案件入力（Step 2）
- テーブル形式で案件を1行ずつ入力できる
- 「＋ 案件を追加」ボタンで行を動的に追加できる
- 各行に削除ボタンを設け、不要な行を削除できる
- 案件名が空の行はレポート出力時に自動除外する

### FR-04 育成方針入力（Step 3）
- 重点スキル領域・育成対象層・目指す人材像・課題・方針を入力できる
- 育成対象層はプルダウン選択とする
- すべて任意入力とする

### FR-05 メンバー別育成目標入力（Step 4）
- テーブル形式でメンバーを1行ずつ入力できる
- 「＋ メンバーを追加」ボタンで行を動的に追加できる
- 現在Lv・目標Lvは1〜5のプルダウン選択とする
- 施策フィールドは読点・カンマ区切りで複数入力でき、レポート上ではチップ形式で表示する
- 氏名が空の行はレポート出力時に自動除外する

### FR-06 レポート生成
- Step 4 の「レポートを生成」ボタンを押すとHTMLレポートを生成する
- 生成したレポートは同一画面の下部に iframe でプレビュー表示する
- プレビュー表示後、ページは自動的にプレビュー位置にスクロールする

### FR-07 レポートダウンロード
- プレビュー上部の「HTMLをダウンロード」ボタンでファイルを保存できる
- ファイル名は `スキル育成計画_{チーム名}_{年度}.html` の形式とする
- ダウンロードしたHTMLファイルは外部リソースなしで単体で表示できる

### FR-08 レポートの印刷対応
- ダウンロードしたHTMLをブラウザで開いて印刷（Ctrl+P）したとき、
  体裁が崩れないよう `@media print` スタイルを実装する

## 4. 非機能要件

### NFR-01 配布・起動の容易さ（最重要）
- HTMLファイル1つを配布するだけで使用できること
- Webサーバー・Python・Node.js 等のインストールが不要なこと
- `file://` プロトコルでブラウザから直接開いて動作すること

### NFR-02 セキュリティ
- APIキーや認証情報を一切含まないこと
- 外部サーバーへのデータ送信を行わないこと
- ユーザー入力値をHTMLに埋め込む前にXSSサニタイズを行うこと
- `localStorage` / `sessionStorage` を使用しないこと（個人情報の意図しない残留防止）

### NFR-03 パフォーマンス
- ファイルを開いてから操作可能になるまで1秒以内であること
- レポート生成からプレビュー表示まで1秒以内であること

### NFR-04 ブラウザ対応
- Chrome 最新版で完全動作すること（必須）
- Edge 最新版で完全動作すること（必須）
- Firefox・Safari 最新版で動作すること（推奨）
- Internet Explorer は対象外

### NFR-05 保守性
- 単一HTMLファイルでの完結を維持すること
- 外部フレームワーク・CDNへの依存を持ち込まないこと
- フォーム項目の追加・削除がコードの局所的な変更で完結すること

### NFR-06 データ容量
- ファイルサイズは圧縮なしで 300KB 以内を目安とする
- 入力データが大量になっても（メンバー50名・案件20件）動作が重くならないこと

## 5. 制約条件

| 制約 | 内容 | 理由 |
|------|------|------|
| サーバー不可 | Webサーバーを立てない | 展開コスト削減・IT管理者不要 |
| API不可 | 外部APIを呼び出さない | インターネット環境に依存しない |
| フレームワーク不可 | React/Vue等を使わない | ファイル単体配布の維持 |
| 外部CDN不可 | jsDelivr等を参照しない | オフライン環境での動作保証 |
| localStorage不可 | ブラウザストレージ不使用 | 個人情報の残留リスク回避 |

---

## 6. 受け入れ基準

以下をすべて満たすことでv1.0完成とみなす。

### 機能面
- [ ] HTMLファイルをダブルクリックするとブラウザで開く
- [ ] Step1〜4の入力・ステップ遷移が正常に動作する
- [ ] 必須項目未入力時に適切なエラーメッセージが表示される
- [ ] 案件・メンバーの行追加・削除が正常に動作する
- [ ] 「レポートを生成」でプレビューが表示される
- [ ] 「HTMLをダウンロード」でファイルが保存される
- [ ] ダウンロードしたHTMLが外部リソースなしで正常に表示される
- [ ] ダウンロードしたHTMLを印刷したとき体裁が崩れない

### セキュリティ面
- [ ] `<script>` タグを含む入力をしてもXSSが発生しない
- [ ] ブラウザの開発者ツールのNetworkタブで外部通信が発生していない

### 品質面
- [ ] Chrome・Edgeで表示崩れがない
- [ ] メンバー20名・案件10件のデータでレポート生成が1秒以内に完了する
- [ ] ファイルサイズが300KB以内である

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| スキルレベル（Lv） | Lv1（指導下で実施）〜Lv5（組織横断で牽引）の5段階評価 |
| 育成施策 | OJT・社外研修・資格取得など具体的な育成アクション |
| 重要度 | 案件遂行上の戦略的重要度。高／中／低の3段階 |
| チップ表示 | タグ形式の小さな丸みを帯びたラベルUI |
| スタンドアロンHTML | 外部ファイル参照なしで単体表示できるHTMLファイル |
